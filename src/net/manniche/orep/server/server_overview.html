Introduction

[Description]

The OREP server architechture provides a very minimal core support of the central operations needed for all server implementations. No clients will have direct contact with the core server, but will use one of the server implementations or shells, that are distributed with the server code:

  1. The JMS server implementation provides access to the core server through a Java Message Queue implementation.
  2. The RMI server implementation provides transparent synchronouos access to the core server through the java Remote Method Invocation protocol. An example client can be found in the net.manniche.orep.client.rmi package.

Overview

[Description]

The core server architecture is composed of an interface ObjectRepository, which defines the core operations that the server supports, and an interface ServiceLocator, that defines the services that can be registered with the server.
